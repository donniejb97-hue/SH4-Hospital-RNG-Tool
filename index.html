THE PROPOSED PROBLEM

In Silent Hill 4's Hospital World, players must locate two items hidden behind randomized doors: the Hospital Key and Eileen's Room. The second-floor corridor stretches out with 11 doors on each side — 22 total — and the game randomly assigns which two doors contain your objectives each playthrough.

There's no way to predict which doors hold the key items. You enter the corridor, pick a direction, and start checking doors one by one. Each wrong door costs seconds of time — the animation to open, peek inside, close, and move to the next one adds up. In a bad run, you might check 15+ doors before finding both items. In a lucky run, they're right next to each other at the front end of the hall.

"Hospital RNG can make or break a speedrun. Bad luck means checking door after door while your time bleeds away. Good luck means walking straight to both items."

02

THE PROPOSED SOLUTION

The Hospital RNG program attaches to the game process, reads the value of a specified memory address, and outputs which doors contain the Hospital Key and Eileen's Room. No more guessing. The moment you enter the hospital world, you know exactly where to go.

HOW IT WORKS

The tool uses memory reading to access the game's door state array. When Silent Hill 4 loads the hospital world, it populates a 22-byte array where each byte represents a door's contents.

Memory Address

The door contents are stored at a fixed memory location that persists across game sessions.

0x13D7B40 (22 bytes)

Value Detection

Each door byte contains a value indicating what's inside. The program scans for specific values.

Hospital Key = 14
Eileen's Room = 10

Index Mapping

The array indices are reversed on both sides of the corridor — a quirk discovered through testing.

Index 0 = L11 → Index 10 = L1
Index 11 = R11 → Index 21 = R1

Real-Time Polling

The program continuously monitors the memory address and updates the display as soon as values are detected.

Update interval: 500ms
